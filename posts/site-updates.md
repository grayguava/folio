# Site updates — final polishes

**Apr 30, 2026**

after getting the basic static site working, i spent the last day on the details. here's what changed:

## Markdown experience

- code blocks now have syntax highlighting in both light and dark modes
- copy button stays fixed in the corner, no longer scrolls with the code
- typography tuned — headings are clearer, body text is more readable
- blog posts show reading time (auto-calculated from word count + code blocks)

## Mobile refinements

- site scales down cleanly on smaller screens
- blog list simplifies on mobile: date, title, no clutter
- code blocks get slightly larger font for readability
- footer quotes hide on mobile to save space

## Visual tweaks

- theme toggle now uses proper sun/moon icons instead of characters
- 404 page respects your theme preference
- content width narrowed slightly for better reading experience
- custom text selection highlight (shows accent color based on dark/light mode instead of default blue)
- elsewhere links now use icons instead of text

## The small stuff

- titles update based on current page
- scramble animation speed adjusted
- quote system simplified — no weights, just random pick, loads once per session
- removed unnecessary complexity throughout

## Why it matters

this site started as an escape from overengineered frameworks. but "simple" can become "rough" if you're not careful. these updates are about making it feel finished without adding back what we escaped from.

the site is now:
- fast (single html, no build)
- clean (organized css, modular js)
- mine (easy to edit, easy to extend)
